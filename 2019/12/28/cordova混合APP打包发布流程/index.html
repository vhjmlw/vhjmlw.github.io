Cordova混合APP打包发布流程

主要步骤：

一、前端人员打包

1.1 先修改生产环境的地址

  • Android 打包
  • Ios 打包

APP前端使用框架：Cordova、Vue、vue-router、vux
Cordova提供将前端代码打包为apk、ipa文件功能。
Vue、vue-router为前端使用的框架。
vux为前端组件库，提供了前端常用的组件。
业务代码部分使用前端代码编写，与系统交互的部分由Cordova各个插件完成。项目中使用到的Cordova的插件有：
cordova-plugin-camera：读取相册或者调用本地摄像头拍照。
phonegap-plugin-barcodescanner：调用手机摄像头扫描二维码。
cordova-plugin-cache-clear：清除缓存。
cordova-plugin-app-version：读取APP版本号。
cordova-plugin-file-transfer：下载apk文件到本地。
cordova-plugin-file-opener2：apk文件下载到本地后，打开apk文件并安装。
cordova-plugin-getuisdk：个推官方Cordova插件，实现消息推送。
cordova-plugin-statusbar：仅iOS端使用，定义状态栏样式，解决部分iOS系统状态栏覆盖APP页面问题。

config.xml文件为Cordova配置文件，里面包含了APP的基本信息，如：APP名称、版本号、包名、桌面图标icon等。

package.json文件为项目的配置文件，里面包含了项目的依赖、打包指令、Cordova插件等信息。
如果项目中还没有node_modules依赖文件夹，说明项目还没有安装依赖。需要执行指令 npm install 先安装依赖，安装完成后项目根目录会自动生成 node-modules 依赖文件夹，里面包含了项目中使用到的依赖。

1.2 Android端打包：

Android端打包的两个情形：

两个不同的情形对应package.json配置文件里面两个不同的指令，根据不同的情形使用对应的指令。
1、正式发布release打包（npm run build-android-release:app）：
使用vhjmlw.keystore秘钥库签名
秘钥库路径：/res/vhjmlw.keystore
秘钥库密码：9294255041
秘钥库秘钥别名：vhjmlw_UIOT
秘钥库秘钥密码：lei9294255041

2、测试debug打包（npm run build-android-debug:app）：

正式发布打包后apk安装包路径：
未签名安装包：/platforms/android/app/build/outputs/apk/release/app-release-unsigned.apk
已签名安装包：/platforms/android/app/build/outputs/apk/release/app-release.apk

测试debug打包后apk安装包路径：
debug签名安装包：/platforms/android/app/build/outputs/apk/debug/app-debug.apk

debug签名的安装包不应该放在线上生产环境，只适合测试用，debug签名的安装包不能上传应用市场，且不同电脑上的debug.keystore秘钥库可能不同，不同电脑上打出来的debug安装包签名可能不同，后期APP更新时签名不同会造成新版本APP无法安装，这样的话可能会造成APP无法更新。

/res/vhjmlw.keystore为公共秘钥库，每次打包时使用此秘钥库打出来的apk安装包签名都一致。

线上发布应该使用/platforms/android/app/build/outputs/apk/release/app-release.apk已签名安装包。
